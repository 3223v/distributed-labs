# Week 4：Snapshot + Version + CAS

## 本周目标

解决 Week 3 的核心痛点：**WAL 无限增长**。Put 10000 次 → WAL 10000 行 → 重启 replay 10000 次 → 越来越慢。

Snapshot 的思路：**定期把内存全量状态拍一张快照写入磁盘，然后把 WAL 截断**。重启时先加载 snapshot，再 replay 快照之后的新增 WAL 记录。

附带做两个小功能：给每个 key 加版本号（version），实现 CAS（Compare-And-Swap）条件更新。

---

## 你应该达到的水平

1. **为什么要做 snapshot？** WAL 的 replay 时间是 O(写入次数)，snapshot + WAL = O(1) 恢复。
2. **snapshot 的原子写入流程是什么？** .tmp → fsync → rename → fsync 目录（防止崩溃时 snapshot 损坏）。
3. **snapshot 为什么必须包含 client_table？** 如果去重表不进快照，重启后忘记处理过的请求，client 重试会导致重复执行。
4. **CAS 的用途是什么？** 并发场景下：只有版本匹配才写入，防止覆盖别人的修改。

### 第一步：`storage/snapshot.py`

```python
class Snapshot:
    @staticmethod
    async def save(path: str, data: dict) -> None:
        """原子保存快照：写 .tmp → fsync → rename → fsync 目录"""

    @staticmethod
    async def load(path: str) -> Optional[dict]:
        """加载快照，文件不存在返回 None"""
```

**Snapshot 内容格式**：
```json
{
    "last_applied": 1024,
    "kv": {
        "x": {"data": "1", "version": 3},
        "y": {"data": "hello", "version": 1}
    },
    "client_table": {
        "c1": {"last_seq": 12, "last_result": {"ok": true, "result": "OK"}}
    }
}
```

注意每条 value 变成了 `{"data": ..., "version": N}`，不再是裸字符串。

**原子写入四个步骤**（缺一不可）：
```
1. 写 snapshot.tmp
2. fsync snapshot.tmp     ← 确保数据落盘
3. rename snapshot.tmp → snapshot.dat
4. fsync 目录              ← 确保 rename 落盘
```

### 第二步：改 `kv/store.py`

改动较大，核心变化：

**value 结构升级**：从 `str` 变成 `{"data": str, "version": int}`

**新增方法**：
```python
async def cas(self, key, expected_version, new_value, client_id, seq) -> dict:
    """只有当前版本 == expected_version 时才写入"""

async def snapshot(self) -> None:
    """生成快照 → 截断 WAL"""

async def load_snapshot(self) -> None:
    """从 snapshot 恢复 → replay 后续 WAL"""
```

**Put 流程变化**：
```
1. 写 WAL
2. 检查当前版本：不存在 → version=1；存在 → version = 旧版本+1
3. self.data[key] = {"data": value, "version": new_version}
4. 每 N 次写入自动触发 snapshot（如 1000 次）
```

**Get 流程变化**：
```
返回 data["data"]（对外仍然返回字符串，版本号内部使用）
```

**CAS 流程**：
```
1. 查当前版本
2. 当前版本 != expected_version → 返回 ok=false, error="版本不匹配"
3. 当前版本 == expected_version → 写 WAL → 更新数据 + 版本+1
```

**启动恢复流程变化**：
```
1. load_snapshot() → 恢复 kv 和 client_table
2. WAL replay → 只 replay snapshot 之后的新增记录
3. 没有 snapshot → 全量 WAL replay（同 Week 3）
```
